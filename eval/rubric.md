# 提言書の評価チェックリスト（8項目）

各ケースの提言書を以下の8項目で評価する（○=満たす / △=一部 / ×=満たさない）。
「複雑な仕組み」ではなく「出力の質」を評価対象にする。

| # | 項目 | 判定基準 |
|---|---|---|
| 1 | 事実と推測の分離 | 「事実情報の整理」に相談文に無い数字・断定が混入していない |
| 2 | 不足情報の質問化 | 不足情報が経営者への具体的な質問の形になっている（「情報が足りない」で終わらない） |
| 3 | 論点の明示 | 専門AI間の意見の食い違いが隠されず論点として書かれている |
| 4 | 判断の順番 | 「次に考えるべきこと」が実行できる順番で並んでいる |
| 5 | 相談先の具体性 | 「誰に・何を持って行くか」まで書かれている（「専門家に相談」で終わらない） |
| 6 | 経営者が読める文章 | 専門用語に説明があり、そのまま渡せる文章になっている |
| 7 | 相互レビューの反映 | Round 2 の指摘が提言書に反映されている（会議ログと突合） |
| 8 | 結論の押し付け回避 | 「やるべき/やめるべき」の断定でなく、条件付きの判断材料になっている |

### 既知の弱点（次の改善候補）
- case1 #1: 議長が「月商≒年商/12」を暗黙に計算して事実のように書いた。
  → 提言書プロンプトに「相談文に無い数字を出す場合は『推定』と明記」を追加する
- 提言書が長い（経営者向けには2ページ程度への要約版があるとよい）
- 1ターンのみ。経営者の回答を受けた再会議（フォローアップ）は未実装
